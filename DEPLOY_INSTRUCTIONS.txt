The error is: package-lock.json is out of sync with the new package.json

1. Copy the new package.json and .npmrc into your project root

WHY THIS HAPPENED:
──────────────────
Your old package-lock.json had:
  - astro@5.18.2   (but package.json said ^4)
  - react@19.2.7   (but package.json said ^18)
  - zod@4.4.3      (but package.json said ^3)
  - @astrojs/cloudflare MISSING from lock

`npm ci` on Cloudflare fails if ANY package in the lock file
doesn't match what package.json specifies.

Deleting the lock and running `npm install` locally creates a fresh
lock file that exactly matches the new package.json.
